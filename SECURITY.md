# 安全政策 / Security Policy

## 报告漏洞

如果你发现安全漏洞，**请不要通过公开 Issue 报告**。

请发送邮件至 **security@xtocn.com**，我们将在 48 小时内确认收到并开始处理。

邮件应包含：
- 漏洞类型与影响范围
- 复现步骤（如适用）
- 建议修复方案（如有）

## 处理流程

1. 收到报告 → 48 小时内确认
2. 验证漏洞 → 评估严重程度
3. 开发修复 → 准备补丁
4. 发布安全公告 + 补丁
5. 在 CHANGELOG 中记录（不透露细节直到多数用户已更新）

## 支持的版本

| 版本 | 支持状态 |
|------|---------|
| 最新 release | ✅ 安全更新 |
| 旧版本 | ❌ 请升级到最新版 |

## 依赖安全

本项目使用 `npm audit` 定期扫描依赖漏洞。如发现高危依赖，会尽快升级或替换。
